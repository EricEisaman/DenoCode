Deno vs Node

Deno is a modern, fast, secure, run time. Node has npm

Before we get in to performance benchmarks lets look at what Deno improves upon and some of it's shortcommings.

✔️ Deno is fast (see benchmarks)
✔️ Deno is secure limiting access to outside environment as standard
✔️ Deno is lightweight, fixes node_modules bloat
✔️ Deno is decentralized, no central package manager
✔️ Deno Async functionality is great
✔️ Deno has more focus on Typescript
✔️ Deno feels up-to-date with less compromises
❌ Deno community is growing but perhaps only 1% currently of Node.js
❌ Less code to copy off stackoverflow
❌ No package manager like npm
❌ Less modules and libraries compared to npm
❌ Developing fast, inevitably some bugs

Ultimately you can do with Deno what you can do with Node. If you want to build server-side code with Javascript or Typescript then both will do the job. Node.js isn't going anywhere, it's hugely popular and that isn't going to change overnight. While Deno does offer some improvements the majority of developers will take some time to start using Deno as default.

So which should I choose for my next project Deno or Node.js?

This ultimately comes down to how much work you'll be doing with external modules and libraries. If you are writing something that mashes together different 3rd party services then Node.js is the obvious choice. If you are writing something where you only need basic module support and that is available via the Deno Modules Directory then Deno would be the obvious choice.
